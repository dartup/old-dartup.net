Dart shared hosting alpha.

Requirements for alpha access:

  • A public git url to download your project from. It may be from any service like GitHub or Bitbucket. The only requirement is that git must be able to clone it wit no authentication.
  • It must use the dartup package so that I can change how the app and the service interact.
  • The main file must bin/server.dart.
  • The main file must work without arguments.
  • And the project can't use any build tools, nothing but a simple pub get. (You can work around that by starting subprocesses from sever.dart if you really want to.)
  • The service will update your code whenever it feels like it.
  • It will try to be up but will fail miserable.
All or most of these requirements will fall away in time. But to get a quick start your app must adapt to the platform.

How to sign up.

You only need to contact me with the following information:
  • The name you want for your project. name.dartup.io.
  • The git url to your project.
  • The main file must bin/server.dart.
  • Whether it's good or evil. See below.
  • Any environmental variables you want to pass along to your application. Like database credentials.
  • If it's evil a short description on the horror it will unleash.

How to sign up.

For good applications I am look for what types of applications you might want to run in a shared host environment. The main focus for me is to get some more real world application to test the platform with. I want to spend my time making the platform not making fake application that may or may not be relevant for what you want. The second is that seeing what you're actually want to run will inform the design and choices so that Dartup can be find the correct balance many apps on a box and useful performance.
A few notes for good apps:
  • Don’t put any code on Dartup that is in any way secret.
  • Don’t link your application to any data you are afraid of or is private.
  • In short Dartup alpha will be a hostile environment for your application. Se evil.
  • Play nice with the other applications on server and the box it self. If you want to be evil then make an evil app. You can have both.
I am also looking for evil application to test and make the platform more resilient against bad actors and bad days. I know that when i was a young programmer writing php applications in productions it happened that i did some horrible stuff. (It was an accident i swear.) So Dartup need some ways to protect itself against all sorts of bad stuff. Evil apps will only be runned when i want to test the platform so do not hold back, 1 sec and the instance crashes is a good test.
Ideas for evil stuff:
  • Try an use up one shared resource. (I am looking at you forkbomb.)
  • Try to get data or information from other apps running on the same box.
  • Try to take over the box in some way.
  • Any other evil thing that will make it a sad day for the other users of your box.
But there will be some rules:
  • You must limit your actions to the box you are running on. To run network security tests I need special permissions from Amazon. So later perhaps.
  • If you take control of the box or other apps. Do not do anything more serious than (Haha got you.)
Breaking of these rules will blacklist you for any future Dartup alpha or beta tests.

Q&A

What databases/datastores are supported?
None yet. But you can host mongoDB for free over at https://mongolab.com/ and if you select Amazone Web Servies eu-west-1 Irland region the database will be nice and close to Dartup alpha. If you search around you will find lots of services that offer free and or cheap database hosting on amazon. And there is always firebase.

I do not feel comfortable just telling you my database credentials. Then don’t. This extremely early alpha and that there is a good hint that your application is not meant for Dartup yet. I hope to see your business in the future when the platform is a bit more secure and stable.

What do i do if i want to change some settings?
Contact me. It serves me right for not having a control panel yet.

Do I have any file access?
Yes and no. You have full read write access to your own home directory. But what you store is not persistent over time. I will look into it but in the short term there will be a lot of nuking.
